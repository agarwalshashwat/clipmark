# Clipmark Navigation Strategy: The Digital Curator's Map This document outlines the navigation architecture for the Clipmark ecosystem, maintaining the "Digital Curator" editorial aesthetic across all touchpoints. --- ## 1. Marketing Navigation (`clipmark.com`) **Goal:** Conversion, Education, and Trust. The navigation should be minimal, airy, and guide the user toward the "Get Started" or "Install Extension" action. ### Primary Header (Lander) * **Brand:** Clipmark Logo (Signature Teal) * **Left/Center Links:** * **Features:** Anchors to the workflow sections (Bookmark, AI, Revisit). * **Pricing:** Direct link to the tier comparison. * **Testimonials:** Social proof for students/creators. * **Right Actions:** * **Log In:** Ghost button for existing users. * **Get Started / Install Extension:** Primary CTA (Solid Teal). ### Footer (Universal) * **Column 1: Product:** Features, Pricing, Download. * **Column 2: Company:** About, Contact, Twitter/X. * **Column 3: Legal:** Privacy Policy, Terms of Service. * **Bottom Bar:** Copyright, Theme Toggle (Light/Dark). --- ## 2. Product Navigation (`app.clipmark.com`) **Goal:** Efficiency, Organization, and Content Discovery. The app requires a persistent sidebar for deep navigation and a focused top bar for context. ### Side Navigation (The Library) * **Top Section:** * **All Bookmarks:** The "Knowledge Stream" (Timeline view). * **Videos:** Grouped view (The cinematic dashboard). * **Revisit Queue:** Spaced repetition tasks for today. * **Middle Section (Curations):** * **Groups:** User-defined folders/playlists. * **Shared:** Collections that have been published. * **Bottom Section:** * **Upgrade to Pro:** High-visibility entry point for free users. * **Settings:** Extension options and account management. ### Top Context Bar * **Breadcrumbs:** `Library > Tutorials > System Design` * **Search:** Universal search for notes, tags, or video titles. * **User Profile:** Avatar, Plan Status (Free/Pro), Log Out. --- ## 3. Extension Navigation (Popup & Side Panel) **Goal:** Immediate Utility and Seamless Capture. Consistency between these views is critical as per our previous design turn. ### Persistent Tabs (Bottom) 1. **Save (Active View):** The current capture interface. 2. **Auto-AI:** Quick access to transcript-based summaries. 3. **Library/Dashboard:** Quick link to the full web app. --- ## 4. Shared View Navigation (`/v/{shareId}`) **Goal:** Consumption and Viral Growth. The navigation here is a hybrid: giving the visitor content while offering them the product. ### Header * **Brand:** Clipmark (Link to lander). * **Secondary Action:** "Save this Collection" (Requires log-in). * **Primary Action:** "Get Extension" (Conversion). --- ## Next Steps 1. **Apply to Designs:** Update the existing Landing Page and Dashboard headers to match this plan. 2. **Breadcrumb Implementation:** Add contextual breadcrumbs to the Dashboard views. 3. **Search State:** Design the "Search Overlay" for the app.
